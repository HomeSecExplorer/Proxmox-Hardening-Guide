# Contributing Guide

Thanks for helping improve the **Proxmox Hardening Guides**!

## Workflow

1. **Search issues first** - avoid duplicates.
1. Fork, create a feature branch, then open a PR.
1. One logical change per PR.
1. Follow the existing typographical conventions.
1. Remember to add an entry to the **Change Notes**.

### Commit Style

`docs-PBS3: clarify verify-new option`

### DCO / Sign‑off

All commits must be signed:

```bash
git commit -s -m "docs-PBS3: clarify verify-new option"
```
